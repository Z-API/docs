---
title: "Enviar múltiplos contatos"
api: "POST /instances/{instanceId}/token/{token}/send-contacts"
description: "Envia múltiplos contatos para um destinatário no WhatsApp"
---

## Conceituação

Simples e objetivo este método permite você enviar vários contatos, você não precisa ter ele em seus contatos, basta preencher os atributos do método com informações do contato e enviar.

## Atributos

### Header

<ParamField header="Client-Token" type="string">
  Token de segurança da conta
</ParamField>

### Obrigatórios

<ParamField body="phone" type="string" required>
  Telefone (ou ID do grupo para casos de envio para grupos) do destinatário no formato DDI DDD NÚMERO. Ex: 551199999999
</ParamField>

<ParamField body="contacts" type="array" required>
  Array de contatos a serem enviados
  <Expandable title="Propriedades">
    <ParamField body="name" type="string" required>
      Nome do contato
    </ParamField>
    <ParamField body="phones" type="array" required>
      Array de números de telefone do contato
    </ParamField>
    <ParamField body="businessDescription" type="string">
      Breve descrição do contato
    </ParamField>
  </Expandable>
</ParamField>

### Opcionais

<ParamField body="messageId" type="string">
  Atributo utilizado para responder uma mensagem do chat
</ParamField>

<ParamField body="delayMessage" type="number">
  Delay adicionado na mensagem. Você pode decidir entre um range de 1 a 15 sec. O delay default caso não seja informado é de 1 a 3 sec
</ParamField>

---

## Request Body

```json
{
  "phone": "5511999999999",
  "contacts": [
    {
      "name": "Nome do contato",
      "phones": ["5544999999999", "5544888888888"]
    },
    {
      "name": "Nome do contato",
      "phones": ["5544999999999"]
    },
    {
      "name": "Nome do contato",
      "businessDescription": "Uma empresa do Grupo Irrah",
      "phones": ["5544999999999"]
    }
  ]
}
```

---

## Response

### 200

```json
{
  "zaapId": "3999984263738042930CD6ECDE9VDWSA",
  "messageId": "D241XXXX732339502B68",
  "id": "D241XXXX732339502B68"
}
```

### 405

Neste caso certifique que esteja enviando corretamente a especificação do método, ou seja verifique se você enviou o POST ou GET conforme especificado no início deste tópico.

### 415

Caso você receba um erro 415, certifique de adicionar na headers da requisição o "Content-Type" do objeto que você está enviando, em sua grande maioria "application/json".
