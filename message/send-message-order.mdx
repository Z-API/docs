## Conceituação

Neste método você poderá enviar mensagens de pedidos, contendo produtos do seu catálogo ou customizados no momento do envio. Esta mensagem é a mesma que é enviada quando você clica no botão "Aceitar pedido" ou "Enviar cobrança".

<Info>
Disponível apenas para contas **business** do WhatsApp. Produtos sem `productId` são tratados como itens customizados.
</Info>

## Atributos

### Header

<ParamField header="Client-Token" type="string">
  Token de segurança da conta
</ParamField>

### Obrigatórios

<ParamField body="phone" type="string" required>
  Telefone do destinatário no formato DDI DDD NÚMERO. Ex: 551199999999
</ParamField>

<ParamField body="order" type="object" required>
  Dados do pedido
  <Expandable title="Propriedades">
    <ParamField body="currency" type="string" required>
      Código da moeda (ex: BRL)
    </ParamField>
    <ParamField body="products" type="array" required>
      Informações dos produtos relacionados ao pedido
      <Expandable title="Propriedades">
        <ParamField body="name" type="string" required>
          Nome do produto
        </ParamField>
        <ParamField body="value" type="number" required>
          Valor do produto
        </ParamField>
        <ParamField body="quantity" type="number" required>
          Quantidade
        </ParamField>
        <ParamField body="productId" type="string">
          ID do produto do catálogo
        </ParamField>
      </Expandable>
    </ParamField>
    <ParamField body="discount" type="number">
      Valor de desconto
    </ParamField>
    <ParamField body="tax" type="number">
      Valor de imposto
    </ParamField>
    <ParamField body="shipping" type="number">
      Valor de frete
    </ParamField>
  </Expandable>
</ParamField>

### Opcionais

<ParamField body="paymentSettings" type="object">
  Configurações de pagamento
  <Expandable title="Propriedades">
    <ParamField body="pix" type="object">
      Configurações do PIX
      <Expandable title="Propriedades">
        <ParamField body="key" type="string">
          Chave PIX
        </ParamField>
        <ParamField body="keyType" type="string">
          Tipo da chave: `cpf`, `cnpj`, `phone`, `email` ou `randomKey`
        </ParamField>
        <ParamField body="name" type="string">
          Nome da chave
        </ParamField>
      </Expandable>
    </ParamField>
    <ParamField body="card" type="object">
      Configurações de cartão
      <Expandable title="Propriedades">
        <ParamField body="enabled" type="boolean">
          Habilitar pagamento por cartão
        </ParamField>
      </Expandable>
    </ParamField>
  </Expandable>
</ParamField>

## Request Body

**Com produtos do catálogo e pagamento:**

```json
{
  "phone": "554499999999",
  "order": {
    "currency": "BRL",
    "discount": 10,
    "tax": 10,
    "shipping": 5,
    "products": [
      {
        "productId": "23940797548900636",
        "name": "Nome do produto",
        "value": 10,
        "quantity": 2
      }
    ]
  },
  "paymentSettings": {
    "pix": {
      "key": "Chave PIX",
      "keyType": "cpf",
      "name": "Nome da chave"
    },
    "card": {
      "enabled": true
    }
  }
}
```

**Apenas campos obrigatórios (produto customizado):**

```json
{
  "phone": "554499999999",
  "order": {
    "currency": "BRL",
    "products": [
      {
        "name": "Nome do produto",
        "value": 150,
        "quantity": 1
      }
    ]
  }
}
```

## Response

### 200

```json
{
  "zaapId": "3999984263738042930CD6ECDE9VDWSA",
  "messageId": "D241XXXX732339502B68",
  "id": "D241XXXX732339502B68"
}
```

### 405

Neste caso certifique que esteja enviando corretamente a especificação do método, ou seja verifique se você enviou o POST ou GET conforme especificado no início deste tópico.

### 415

Caso você receba um erro 415, certifique de adicionar na headers da requisição o "Content-Type" do objeto que você está enviando, em sua grande maioria "application/json".
