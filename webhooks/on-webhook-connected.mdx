## Conceituação

Esse é o webhook de retorno de conexão do celular com o Z-API. É acionado quando o Z-API se conecta ao WhatsApp — durante a leitura do QR code, reinício da instância, etc.

<Warning>
O Z-API não aceita webhooks que não sejam **HTTPS**.
</Warning>

---

## Atualizar webhook

### Header

<InstanceId />
<Token />
### Body

<ParamField body="value" type="string" required>
  URL do webhook de conexão
</ParamField>

### Request Body

```json
{
  "value": "https://endereco-do-seu-sistema.com.br/instancia/SUA_INSTANCIA/connected"
}
```

---

## Retorno do webhook

<ResponseField name="type" type="string">
  Tipo do evento: `ConnectedCallback`
</ResponseField>

<ResponseField name="connected" type="boolean">
  Status de conexão da instância
</ResponseField>

<ResponseField name="phone" type="string">
  Número de telefone conectado
</ResponseField>

<ResponseField name="momment" type="integer">
  Timestamp da conexão
</ResponseField>

<ResponseField name="instanceId" type="string">
  Identificador da instância
</ResponseField>

```json
{
  "type": "ConnectedCallback",
  "connected": true,
  "momment": 26151515154,
  "instanceId": "instance.id",
  "phone": "5544999999999"
}
```

---

## Response

### 200

```json
{
  "value": true
}
```

### 405

Neste caso certifique que esteja enviando corretamente a especificação do método, ou seja verifique se você enviou o POST ou PUT conforme especificado no início deste tópico.

### 415

Caso você receba um erro 415, certifique de adicionar na headers da requisição o "Content-Type" do objeto que você está enviando, em sua grande maioria "application/json".
