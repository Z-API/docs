## Conceituação

Método utilizado para confirmar o código que você recebeu. Para utilizar esse método, você precisa concluir as etapas de registro anteriores, que envolvem verificar a disponibilidade de registro do número e solicitar o código de confirmação.

---

## Atributos

### Header

<InstanceId />
<Token />


<ParamField header="Client-Token" type="string">
  Token de segurança da conta
</ParamField>

### Body

<ParamField body="code" type="string" required>
  Código de confirmação
</ParamField>

---

## Request Body

```json
{
  "code": "123456"
}
```

---

## Response

### 200

<ResponseField name="success" type="boolean">
  Retorna true caso o código tenha sido confirmado corretamente. Feito isso, a instância estará conectada.
</ResponseField>

<ResponseField name="confirmSecurityCode" type="boolean">
  Retorna true se for necessário a confirmação do código de verificação em duas etapas.
</ResponseField>

**Sucesso:**

```json
{
  "success": true
}
```

**Necessário confirmar código de segurança:**

```json
{
  "success": false,
  "confirmSecurityCode": true
}
```

### 400

Requisição inválida. Verifique se os dados que você está enviando estão de acordo com o documentado.

### 405

Neste caso certifique que esteja enviando corretamente a especificação do método, ou seja verifique se você enviou o POST ou GET conforme especificado no início deste tópico.

### 415

Caso você receba um erro 415, certifique de adicionar na headers da requisição o "Content-Type" do objeto que você está enviando, em sua grande maioria "application/json".
