## Conceituação

Método utilizado para atualizar a URL do webhook onde serão recebidos os callbacks de falha na conexão com o proxy.

---

## Parâmetros

<InstanceId />
<Token />

## Atributos

<ParamField body="value" type="string" required>
  URL do webhook que receberá os callbacks de falha.
</ParamField>

---

## Request Body

```json
{
  "value": "http://localhost:3001/api/webhook"
}
```

---

## Comportamento do callback

Caso haja erro ao se conectar utilizando o proxy configurado, serão feitas **3 tentativas de retry**.

Caso a conexão falhe na terceira tentativa, a instância é conectada **sem utilizar nenhum proxy**, e um webhook é disparado para a URL configurada neste endpoint, com o seguinte payload:

```json
{
  "type": "ProxyCallback",
  "instanceId": "3F2XXXXXXXXXXXXXXXXXXXA2FF",
  "error": "The instance was connected without proxy, the configured proxy (socks5://user:senha123@localhost:1080) failed 3 times to connect."
}
```

---

## Response

### 200

```json
{
  "value": true
}
```

### 405

Neste caso certifique que esteja enviando corretamente a especificação do método, ou seja verifique se você enviou o PUT conforme especificado no início deste tópico.

### 415

Caso você receba um erro 415, certifique de adicionar na headers da requisição o "Content-Type" do objeto que você está enviando, em sua grande maioria "application/json".
