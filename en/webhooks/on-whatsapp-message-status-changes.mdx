## Overview

This is the message status callback webhook.

<Warning>
Z-API does not accept webhooks that are not **HTTPS**.
</Warning>

---

## Update webhook

### Header

<InstanceId />
<Token />
### Body

<ParamField body="value" type="string" required>
  Message status webhook URL
</ParamField>

### Request Body

```json
{
  "value": "https://your-system-address.com/instance/YOUR_INSTANCE/status"
}
```

---

## Webhook callback

<ResponseField name="instanceId" type="string">
  Instance identifier
</ResponseField>

<ResponseField name="status" type="string">
  Message status: `SENT`, `RECEIVED`, `READ`, `READ_BY_ME` or `PLAYED`
</ResponseField>

<ResponseField name="ids" type="array">
  Message identifier(s)
</ResponseField>

<ResponseField name="momment" type="integer">
  Event timestamp
</ResponseField>

<ResponseField name="phoneDevice" type="integer">
  Device type (0 = mobile)
</ResponseField>

<ResponseField name="phone" type="string">
  Destination phone number
</ResponseField>

<ResponseField name="type" type="string">
  Event type: `MessageStatusCallback`
</ResponseField>

<ResponseField name="isGroup" type="boolean">
  Whether the chat is a group
</ResponseField>

```json
{
  "instanceId": "instance.id",
  "status": "SENT",
  "ids": ["999999999999999999999"],
  "momment": 1632234645000,
  "phoneDevice": 0,
  "phone": "5544999999999",
  "type": "MessageStatusCallback",
  "isGroup": false
}
```
