## Overview

Method responsible for sending images to your chats. You can work with images in 2 ways:

- **By Link**: if you have the image hosted somewhere on the internet, just send the image link;
- **By Base64**: if you want to send the image in Base64 format.

<Tip>
If you choose Base64, before the binary you need to add the following expression: **data:image/png;base64,** followed by your Base64 code.

Tools to convert images to Base64:
- [base64-image.de](https://www.base64-image.de/)
- [base64.guru](https://base64.guru/converter/encode/image)
</Tip>

## Attributes

### Header

<InstanceId />
<Token />
### Required

<ParamField body="phone" type="string" required>
  Recipient's phone number (or group ID for group messages) in DDI DDD NUMBER format. E.g.: 551199999999
</ParamField>

<ParamField body="image" type="string" required>
  Image link or Base64
</ParamField>

### Optional

<ParamField body="caption" type="string">
  Image caption
</ParamField>

<ParamField body="messageId" type="string">
  Attribute used to reply to a chat message, just add the messageId of the message you want to reply to
</ParamField>

<ParamField body="delayMessage" type="number">
  Delay added to the message. You can choose between a range of 1 to 15 sec, meaning how many seconds it will wait before sending the next message. (E.g.: "delayMessage": 5). The default delay if not provided is 1 to 3 sec
</ParamField>

<ParamField body="viewOnce" type="boolean">
  Whether it will be a view-once message or not
</ParamField>

## Request Body

**By link:**

```json
{
  "phone": "5511999999999",
  "image": "https://www.z-api.io/wp-content/themes/z-api/dist/images/logo.svg",
  "caption": "Logo",
  "viewOnce": false
}
```

**By Base64:**

```json
{
  "phone": "5511999999999",
  "image": "data:image/png;base64,iVBORw0KGgoAAAANSUhEUgAAAyAAAAMgCAIAAABUEpE...",
  "caption": "Logo",
  "viewOnce": false
}
```

### 405

Make sure you are correctly sending the method specification, that is, verify that you sent POST or GET as specified at the beginning of this topic.

### 415

If you receive a 415 error, make sure to add the "Content-Type" header to your request, which in most cases is "application/json".
