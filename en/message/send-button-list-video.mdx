## Overview

With this method you can send videos with action button options, the button content e.g.: YES / No, can be chosen by the user and will be used as a response to the message sent along with the buttons.

<Warning>
Button sending is currently available, but there are some decisive factors for it to work. For more details check the [Button Status](/en/tips/button-status) topic.
</Warning>

<Info>
The **message** attribute cannot be sent empty!
</Info>

---

## Attributes

### Header

<InstanceId />
<Token />
### Required

<ParamField body="phone" type="string" required>
  Recipient's phone number (or group ID for group messages) in DDI DDD NUMBER format. E.g.: 551199999999
</ParamField>

<ParamField body="message" type="string" required>
  Text to be sent
</ParamField>

<ParamField body="buttonList" type="object" required>
  Object with video and button list
  <Expandable title="Properties">
    <ParamField body="video" type="string" required>
      Video URL or Base64
    </ParamField>
    <ParamField body="buttons" type="array" required>
      Array of buttons
      <Expandable title="Properties">
        <ParamField body="label" type="string" required>
          Button text
        </ParamField>
        <ParamField body="id" type="string">
          Button identifier
        </ParamField>
      </Expandable>
    </ParamField>
  </Expandable>
</ParamField>

### Optional

<ParamField body="delayMessage" type="number">
  Delay added to the message. You can choose between a range of 1 to 15 sec, meaning how many seconds it will wait before sending the next message. (E.g.: "delayMessage": 5). The default delay if not provided is 1 to 3 sec
</ParamField>

---

## Request Body

```json
{
  "phone": "5511999999999",
  "message": "Is Z-API Good?",
  "buttonList": {
    "video": "https://example.com/video.mp4",
    "buttons": [
      {
        "id": "1",
        "label": "Great"
      },
      {
        "id": "2",
        "label": "Excellent"
      }
    ]
  }
}
```
